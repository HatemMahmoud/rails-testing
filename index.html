Introduction

Software Testing

The process of analyzing a software item to detect the differences between existing and required conditions.

IEEE

Verification vs. Validation

  • Verification
    Have we built the software right?
  • Validation
    Have we built the right software?

Testing Methods

  • Static vs. Dynamic
  • White-Box vs. Black-Box

Static vs. Dynamic

  • Static Testing
    reviews, walkthroughs, inspections, etc.
  • Dynamic Testing
    executing programmed code with test cases.

White-Box vs. Black-Box

  • White-Box Testing
    Tests internal workings of a software.
  • Black-Box Testing
    Tests the functionality exposed to the end-user.

Testing Philosophies

1. No Testing!

2. Waterfall Testing

Testing is performed by independent group of testers in a separate phase after the implementation.

3. Agile Testing

Testing is performed by all team members incrementally and iteratively.

4. Test-Driven Development (TDD)

For each software unit, a software developer must:

  1. Write a failing test
  2. Write the minimum amount of code to make the test pass
  3. Refactor the new code to acceptable standards

5. Behavior-Driven Development (BDD)

Combines the techniques and principles of TDD with ideas from object-oriented analysis and design to provide software developers and business analysts with a shared process to collaborate on software development.

Rails Testing

  • Testing support was built into Rails from the beginning.
  • Rails creates a "test" folder as soon as you generate an app
  • Rails produces skeleton test code when you generate your models and controllers.
  • The "testing" environment is used when running tests.
  • Fixtures are a way of organizing sample data to populate the testing database before tests run.
